# Nmap

## Состояния UDP-портов в nmap

При UDP-сканировании (`-sU`) nmap отправляет пустой UDP-пакет на целевой порт и оценивает ответ:

| Состояние | Условие | Значение |
|-----------|---------|----------|
| `open` | Получен UDP-ответ (или ответ протокола) | Сервис доступен |
| `open\|filtered` | Ответа нет, ICMP Port Unreachable не получен | Порт либо открыт и молчит, либо отфильтрован брандмауэром |
| `closed` | Получен ICMP Port Unreachable | Порт закрыт, сервис не слушает |

**Важно:** Для UDP `open|filtered` — это не гарантия, что порт открыт. nmap не может отличить молчащий порт от отфильтрованного. Окончательный вывод делается на основе конфигурации устройства или анализа трафика.

## Как nmap обнаруживает хост перед UDP-сканированием

Перед UDP-сканированием nmap выполняет обнаружение хоста (`-sn` по умолчанию, если не указано `-Pn`). В процессе отправляются:

- ICMP Echo Request
- TCP SYN на порт 443
- TCP ACK на порт 80
- ICMP Timestamp Request

Это может вызвать **RST-ответы** от закрытых TCP-портов (например, 80). **Это не означает, что HTTP-сервер включён** — так работает TCP-стек устройства, отвечая RST на пакеты к незанятым портам.

## UDP-сканирование роутера

**Задача:** проверить с внешней машины, какие UDP-порты роутера доступны и отвечают ли они

### Базовое сканирование подозрительных портов

```bash
# Требует прав root
nmap -sU -p 19,161,162,500,4500,1645,1646,1723 --host-timeout 15s <router_pub_ip>
```

### Проверка SNMP с community public

```bash
nmap -sU -p 161 --script snmp-info --script-args snmp.community=public <router_pub_ip>
```

**Важно:**

- Не используйте `-A`, `-sV` или агрессивные скрипты на портах VPN (500, 4500, 1723) — они инициируют handshake и могут нарушить работу туннелей.
- `open|filtered` в выводе nmap **не означает**, что порт открыт и отвечает. Для UDP это стандартная реакция на отсутствие ответа. Окончательное подтверждение — трассировка в Wireshark.

**Ограничение:** nmap не всегда способен отличить «порт молчит» от «порт отфильтрован». При `open|filtered` доверяйте конфигурации устройства и анализу трафика.

### Wireshark-фильтр для контроля ответов

```
ip.src == <router_pub_ip>
```

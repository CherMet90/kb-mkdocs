##### Проверка активных подписок:
```
show ip igmp groups
```
Можно искать по адресу группы, ip приемника, его вилану и т.д.

#### Траблшутинг (предполагается, что igmp snooping включен)
0. Имеющимися средствами определяем в каком vlan живёт приёмник (ip arp и т.п.)
1. Смотрим есть ли от него Join (reports). Там мы должны видеть ip приемника, адрес группы, с какого порта пришел запрос, нашлись ли источники
```
show ip igmp snooping groups vlan <vlan_id> det     # лучше это делать сразу после инициализации подписки, т.к. если она не удачна, то через no-querier-timer (3 минуты by default) вывод станет пустым
```
2. Если нужная запись есть, переходим к проверке знает ли коммутатор куда слать репорты
```
show ip igmp snooping mrouter vlan <vlan_id>        # в выводе мы должны увидеть порт, в который мы считаем должны уходить igmp reports. Если порт в списке есть, значит коммутатор получает с него PIM Hellos, считает его igmp-активным и будет пересылать в него reports
```
3. Также можно посмотреть счетчики igmp по вилан
```
show ip igmp snooping statistics vlan <vlan_id>
# сброс счетчиков
clear ip igmp snooping statistics vlan <vlan_id>
```
